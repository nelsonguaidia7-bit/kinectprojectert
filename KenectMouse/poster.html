Abstract

This project explores camera-based, touchless input on a Raspberry Pi 4 using the Kinect for Xbox 360 (Kinect v1)—a depth-capable sensor. The goal is an alternative to a physical mouse for desktop control and, later, small interactive games from the same pipeline. On Linux with libfreenect, the system captures depth and optional RGB frames and maps hand or marker motion to cursor movement and clicks.

The stack uses Python, NumPy, OpenCV, and freenect bindings to segment the nearest object or a colored marker, then emits virtual mouse events via /dev/uinput (Docker-friendly) or PyAutoGUI on a graphical desktop. Tunable depth bands, blob limits, and screen margins adapt to different rooms and lighting.

Kinect v1 uses structured-light depth at about 640×480—not the newer Azure Kinect. Roadmap: reliable mouse emulation first; lightweight games once latency and robustness meet your target.

Features
  • Depth cursor: nearest-object or depth-band blob tracking with screen mapping and calibration.
  • Wave / arm: gesture to arm tracking before cursor control.
  • Virtual mouse: Linux uinput (kinect_v1_mouse.py) including Docker on Pi.
  • Desktop path: kinect_mouse.py + PyAutoGUI; optional depth “push” click.
  • RGB marker mode: HSV tracking with optional depth refinement.
  • Preview & capture: live depth/RGB preview and PNG snapshots.
  • DevOps: Dockerfile, docker-compose, WSL2 + usbipd for Windows dev.
  • Planned — Small games: same pipeline for timing, scoring, simple physics.

Kinect capabilities

The Kinect for Xbox 360 delivers a depth map (~640×480 @ ~30 fps) via IR structured light, plus RGB video, tilt motor, and on some units a mic array (not used here).

On Linux with libfreenect, this project uses custom blob/marker tracking—not the Windows Kinect SDK’s skeleton runtime. Full body tracking would need another pipeline (e.g., ML pose); current scripts stay depth/RGB based.
